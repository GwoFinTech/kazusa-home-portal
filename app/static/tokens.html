API 使用说明

创建的 Token 可以以 Authorization: Bearer <token> 的形式，程序化访问所有经过 Traefik kazusa-auth@docker 认证的 *.kazusa.feng.moe 服务。

基本用法

curl -H "Authorization: Bearer hp_xxxxxxxxxxxxxxxxxxx" \
  https://kazusa.feng.moe/api/me

可用端点示例

  • GET /api/me — 获取当前登录用户信息
  • GET /api/services — 列出所有服务及访问权限
  • GET /auth/tokens — 管理自己的 Token

请求头规范

  • 必须携带 Authorization: Bearer <token>
  • 不需要额外的 CSRF Token（Bearer 认证不怕 CSRF）
  • 与浏览器 Cookie 认证互斥，同时只需要一种

安全提示

  • Token 仅在创建时展示一次，请立即复制并妥善保管
  • 如果怀疑泄露，请立即点击「撤销」，所有使用该 Token 的客户端会立即失去访问权限
  • 建议为不同场景创建独立 Token，并设置合理的过期时间
小提示：在脚本或 CI 中使用时，建议将 Token 存入环境变量，避免硬编码。
